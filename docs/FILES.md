# Project File Map - Starfield ESM Parser

This document describes the purpose and role of the files within the `./starfield` directory.

## Root Directory
- `README.md`: Project overview, current status, and roadmap.
- `.gitignore`: Git exclusion rules.
- `Makefile`: Build system for the project.
  - Usage: `make <target>`
  - Common Targets:
    - `make all`: Builds all project binaries.
    - `make sdl_bulk_dump` (etc): Builds a specific tool.
    - `make clean`: Removes all object files and binaries.
    - `make test`: Builds and runs the parser unit tests.
    - `make coverage`: Builds and runs tests with gcov coverage reporting.
- `db.conf`: Database connection parameters (git-ignored).

## /bin (Compiled Binaries)
*Note: These are the compiled results of the source code in `/src`.*
- `sdl_bulk_dump`: High-performance tool to index all files in the Data folder (including assets and plugins embedded in .ba2 archives) into the MySQL database.
  - Usage: `./sdl_bulk_dump <data_directory_or_file>`
  - Description: Recursively scans for game data files and BA2 archives. It registers every asset in the `assets` table and extracts all record metadata and EDIDs from .esm/.esp/.esl files into the `records` and `edids` tables. Uses a Virtual File System (VFS) layer for on-the-fly decompression.
- `sdl_search`: Search tool for finding specific tags or values within ESM files.
  - Usage: `./sdl_search --edid <editorid>`
  - Description: Queries the MySQL database for the record's offset, reads it from the game files, and prints deserialized fields.
- `test_parser`: Executable for running the parser unit tests.
  - Usage: `./test_parser`
  - Description: Runs internal unit tests to verify TLV parsing and schema mapping.
  - Note: The `make test` target manages the building and execution of individual test suites.
- `dump_record`: Utility to extract and display the raw/mapped content of a single record.
  - Usage: `./dump_record <esm_file>`
  - Description: Searches for the first `KYWD` record in the specified file and dumps its binary data.
- `starfield_reader`: The primary tool for reading, traversing, and indexing ESM files.
  - Usage: `./starfield_reader <esm_file> <mode> [offset]`
  - Modes:
    - `--index`: Generates SQL `INSERT` statements for all records with `EDID` tags.
    - `--dump <offset>`: Deserializes and prints the record at the given offset (supports hex or dec).
- `header_dump`: Utility to display the Starfield ESM file header information.
  - Usage: `./header_dump`
  - Description: Dumps the first 512 bytes of the hardcoded `Starfield.esm` path.

## /src (Source Code)
The source is organized into functional layers:

### /src/core (Engine)
- `parser.hpp` / `parser.cpp`: The core engine. Implements the TLV (Tag-Length-Value) parsing logic used to traverse Bethesda's binary record format.
- `vfs.hpp`: Defines the Virtual File System (VFS) interface (`IReadStream`) and implementations for raw files and memory buffers.
- `ba2_stream.hpp`: Implementation of the BA2 archive reader. Handles BA2 header parsing, file table lookups, and zlib decompression.
- `indexer.hpp` / `indexer.cpp`: High-level logic for recursively crawling the Data folder and identifying assets and plugins within BA2 archives.
- `schema.hpp`: The project "dictionary". Defines record signatures and the mapping of 4-character tags to data types. Autogenerated from xEdit definitions.

### /src/db (Persistence)
- `mysql_client.hpp` / `mysql_client.cpp`: Handles MySQL connection, bulk inserts, and database maintenance (TRUNCATE, etc.).
- `idatabase.hpp`: Interface for database operations to allow mocking in unit tests.

### /src/tools (Executables)
- `sdl_bulk_dump.cpp`: Implementation of the high-speed database ingestion tool. Orchestrates the Indexer and MySQLClient.
- `sdl_search.cpp`: Implementation of the search functionality, allowing the user to find records based on specific field criteria.
- `starfield_reader.cpp`: The main entry point for high-level ESM file interaction and data extraction.
- `dump_record.cpp`: Source for the `dump_record` tool; focuses on deserializing a specific record into a human-readable format.
- `header_dump.cpp`: Source for the `header_dump` tool; parses the global file header.

## /test (Testing)
The testing suite mirrors the `/src` layout to provide comprehensive coverage across all layers.

### /test/core (Core Logic)
- `test_parser.cpp`: Unit tests for the TLV parser, including edge-case handling and boundary conditions.
- `test_ba2.cpp`: Tests for the BA2 archive reader, using synthetic archives.
- `test_indexer.cpp`: Tests for the directory crawling and asset identification logic.
- `test_vfs.cpp`: Tests for the VFS read streams (File and Memory).
- `golden_test.esm`: A small, synthetic ESM file used for end-to-end verification of the indexing pipeline.

### /test/db (Database)
- `test_mysql.cpp`: Integration tests for the MySQL client, utilizing a separate `starfield_test` database.
- `test_mysql_edge_cases.cpp`: Targeted tests for DB error handling and batching edge cases.
- `mock_database.hpp`: Mock implementation of `idatabase.hpp` for isolated unit testing.

### /test/tools (Integration)
- `test_integration.cpp`: High-level integration tests that verify `sdl_bulk_dump` and `sdl_search` against both the golden file and real-world assets (e.g., `CwLemonade.esm`).

### /test (Root)
- `smoke_test.cpp`: Minimal test to verify the `doctest` environment and build pipeline.
- `test_schema.cpp`: Utility to verify the generated `schema.hpp` contains expected record types.
- `test_header_size.cpp`: Utility to verify the memory layout of the `RecordHeader` struct.

## /third_party (External Libraries)
- `doctest/doctest.h`: A fast, single-header C++ unit testing framework used for the project's test suite.

## /scripts (Development Scripts)
Helper scripts used for codebase maintenance and bulk data ingestion.

- `parse_schema.py`: Automated schema generator.
  - Usage: `python3 parse_schema.py`
  - Description: Parses `vendor/tes5edit/Core/wbDefinitionsSF1.pas` and generates the complete `src/core/schema.hpp` file.
- `generate_golden.cpp`: Utility to create synthetic ESM files for test verification.
- `starfield_index.sql`: Database seed file for populating the MySQL index.
- `schema_head.txt` / `schema_tail.txt`: Boilerplate fragments used by `parse_schema.py` to wrap the autogenerated schema.
